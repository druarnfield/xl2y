# xl2y — Design

**Date:** 2026-07-22
**Status:** Validated in brainstorming session

## Purpose

A Python library with exactly one purpose: take Excel files, apply a
customisable, reproducible data-cleaning pipeline, export to Parquet.

Two ends of a spectrum, one API:

```python
# piece-of-shit mode: don't make me think
xl2y.load("mystery.xlsx").kitchen_sink().to_parquet("out.parquet")

# careful mode: normalise, pin the schema, validate
xl2y.load("report.xlsx", sheet="Q3").clean().conform(schema).to_parquet("out.parquet")
```

## Decisions made

| Question | Decision |
|---|---|
| Pipeline definition | Python code IS the pipeline. Reproducibility = re-run the script. No config serialization. |
| Custom steps | Any `df -> df` callable via `.apply()`. |
| Backend | Polars. Arrow-native Parquet, low string memory, vectorized coercion. No pandas, no pyarrow dependency. |
| Validation | Own minimal `Schema` object (names, types, nullability, bounds, patterns). Not pandera/patito. |
| Type constructor naming | Trailing underscore on ALL constructors for consistency: `str_`, `int_`, `float_`, `bool_`, `date_`, `datetime_`, `cat_`. |
| Unit of work | Sheet-first (`load()`), thin workbook batch helper (`load_all()`). |
| Validation failure | Raise by default, one exception carrying ALL problems. Lenient modes opt-in. |
| Execution | Eager. Excel parsing at `load()` is the cost; lazy adds complexity for nothing. |

## Public API

Two entry points, seven chainable verbs, one schema object, one patterns module.

### Entry points

- `xl2y.load(path, sheet=None, **hints)` — one table. If `sheet` omitted:
  single real sheet → use it; several → pick the one that extracts the
  biggest/most competent table and log the choice.
- `xl2y.load_all(path, **hints)` — every sheet; the same chained verbs apply
  to each; `to_parquet(dir)` fans out one file per sheet
  (`out/q3_sales.parquet`, ...).

`**hints` carry the loader knobs that exist today: `header_rows`,
`sparse_rows`, `dayfirst`, `skip_hidden`, etc.

### Verbs on `Table`

| Verb | Meaning |
|---|---|
| `.clean()` | Safe, deterministic tidy-up: snake_case headers, strip whitespace, drop empty rows/cols, NA tokens, tolerant type coercion (`"$1,234"`, `"(500)"`, `"15%"`, date strings). |
| `.kitchen_sink()` | `clean()` plus every heuristic: comment/title stripping, multi-row header detection, sparse-row handling, unpivot detection. Runs a candidate tournament and keeps the winner (see below). |
| `.cast(col=type_, ...)` | Explicit dtype control where coercion guessed wrong. |
| `.apply(fn)` | Escape hatch: any `df -> df` callable becomes a step. |
| `.conform(schema, on_error="raise")` | Cast to schema, then validate. See Schema section. |
| `.dry_run()` | Non-terminal peek: prints shape, dtypes, first N rows, and what the pipeline did so far (renames, stripped rows, coercions). Returns the `Table` unchanged so it can sit mid-chain during development. |
| `.to_parquet(path)` / `.collect()` | Terminal: write Parquet (lineage embedded in schema metadata), or return the polars DataFrame. |

Every verb returns a **new** `Table` (immutable), so a reusable pipeline is
just a function: `def pipeline(t): return t.clean().cast(...).conform(schema)`.

## The `Table` object

Thin immutable wrapper around three things:

- `df` — polars DataFrame (current state)
- `lineage` — append-only records of what each verb did: columns renamed
  (with originals), rows removed and why, coercions applied, comments/titles
  stripped. Generalises the existing `comments` concept.
- `source` — path, sheet name, load timestamp

Lineage powers three features cheaply:

1. `dry_run()` pretty-prints it.
2. `to_parquet()` embeds it as JSON in Parquet schema metadata
   (key `xl2y`) — self-describing output files.
3. Error reports can cite **original Excel row numbers**, which is what the
   person fixing the file actually needs (they work in Excel).

After `conform()` with lenient modes: `table.rejects` (quarantined rows),
`table.errors` (report mode).

## kitchen_sink: candidate tournament

Generate a handful of interpretations of the raw grid — 1 vs 2 header rows,
sparse rows as comments vs sections vs data, with/without unpivot — score
each on "competent table" signals, keep the winner:

- header uniqueness (no duplicate/fallback `col_3`-style names)
- dtype consistency down columns
- fill density
- plausible shape (rows ≫ header noise)

Runner-up candidates are recorded in lineage so `dry_run()` can show what was
considered and why the winner won.

## Schema and conform

Deliberately small: a declaration of "what this file is supposed to be," not
a general validation framework.

```python
schema = xl2y.Schema(
    store=xl2y.str_(),
    date=xl2y.date_(),
    revenue=xl2y.float_(nullable=False, min=0),
    state=xl2y.cat_("NSW", "VIC", "QLD"),
    email=xl2y.str_(pattern=patterns.EMAIL),
)
```

- Constructors: `str_`, `int_`, `float_`, `bool_`, `date_`, `datetime_`,
  `cat_(*values)`. Options: `nullable=` (default True), `min=`/`max=`
  (numerics, dates), `pattern=` (strings), `check=` (polars expression or
  callable — the per-column escape hatch).
- Schema-level: `extra_columns="keep" | "drop" | "error"` (default keep);
  missing declared columns are always an error.

`conform(schema)` runs two phases, **collecting** failures across both:

1. **Cast** — bring each declared column to its type using the same tolerant
   parsers as `clean()` (`"$1,234"` casts to `float_` fine).
2. **Validate** — nullability, bounds, patterns, checks.

Then one `SchemaError` with every problem: column, rule, count, sample
offending rows with original Excel row numbers. Machine-readable via
`.problems`.

Failure modes: `on_error="raise"` (default) | `"quarantine"` (bad rows →
`table.rejects`) | `"report"` (keep all, populate `table.errors`).

### patterns module

Shipped: `EMAIL`, `URL`, `UUID`, `IPV4`, `PHONE_AU`, `PHONE_E164`,
`POSTCODE_AU`, `ABN`, `ACN`, `TFN`, `BSB`, `DATE_ISO`, `CURRENCY`.
ABN/ACN/TFN are checksum-validated checks, not just regex.

Composable helpers so common cases never need hand-rolled regex:
`patterns.any_of(...)`, `patterns.exact(...)`, `patterns.digits(n)`,
`patterns.one_of("Y", "N")`. A raw regex string always works too.

## Loading engine and memory profile

Keeps the architecture of the existing `excel_loader.py` (which this library
supersedes), with a polars landing zone:

1. **Structure pass** — streaming `iterparse` of each sheet's XML: merged
   ranges, hidden rows/cols (as intervals), formula cells, and the *true*
   data extent. Defeats dimension bloat: a file claiming 1M rows with 200
   real ones costs 200 rows.
2. **Value pass** — openpyxl read-only streaming, bounded by the extent from
   pass 1, into per-column lists, then polars `Series`. No pandas object
   arrays.
3. **Extraction** — existing heuristics ported: merged-comment detection,
   sparse-row handling, multi-row header building, column-run bounding box.
   Coercion becomes polars expressions (vectorized `_parse_numberish`
   equivalent as chained string ops), not Python-level `.map`.

Memory: O(structure) for pass 1, O(actual table) after; never O(declared
dimensions). `load_all()` processes sheets one at a time, releasing each —
peak = largest sheet, not the sum.

`.xls` / `.xlsb`: explicitly unsupported with a convert-first error message.
Formula cells with no cached value: detected, warned (they read as null).

## Errors

All under `Xl2yError`:

- `UnsupportedFormatError` — .xls/.xlsb, with conversion instructions
- `EmptySheetError`
- `SheetNotFoundError` — lists available sheets
- `SchemaError` — full multi-failure report; `.problems` for machines

Warnings via `logging`. The library never prints except `dry_run()`.

## Testing strategy

- **Fixture workbooks generated in code** (`tests/fixtures.py`, openpyxl):
  dimension bloat, merged banners, multi-row headers, hidden scratch
  columns, uncached formulas, mixed-type columns.
- **Boundary tests per heuristic** (e.g. merge at exactly 50% width).
- **Golden tests for `kitchen_sink()`**: nasty fixture in, assert winning
  candidate's shape and columns.
- **Bounded-resources test**: a 1M-row-declared bloated file must load fast.

## Package layout

```
xl2y/
  __init__.py      # load, load_all, Schema, type constructors, patterns
  table.py         # Table, lineage, verbs
  reader.py        # structure pass + value pass (ported from excel_loader)
  extract.py       # table extraction heuristics
  coerce.py        # polars-native coercion expressions
  schema.py        # Schema, types, conform
  patterns.py      # EMAIL, PHONE_AU, helpers
  sink.py          # kitchen_sink candidate tournament + scoring
```

Dependencies: `polars`, `openpyxl`. Nothing else (polars writes Parquet
natively).

CLI later: `python -m xl2y file.xlsx` as a thin wrapper over
`load_all().kitchen_sink().to_parquet()`.
